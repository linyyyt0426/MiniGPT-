# MiniGPT 古诗生成器

一个基于 PyTorch 实现的轻量级 Transformer 语言模型，专门用于学习并生成中国古典五言/七言绝句。

---

## 项目概述

本项目实现了一个字符级别的 GPT 风格 Transformer 模型（MiniGPT），使用 24 首经典唐诗作为训练语料。模型通过学习古诗中的字符序列规律，能够根据用户输入的上下文续写诗句，生成符合古诗韵律的文本。

### 核心特性

- **轻量级架构**：6 层 Transformer，4 头注意力，128 维嵌入
- **字符级建模**：基于汉字字符的端到端生成，无需分词
- **交互式对话**：支持实时输入上下文，模型自动续写诗句
- **智能终止**：生成到逗号（，）或句号（。）时自动停止，保证诗句完整性

---

## 技术架构

### 模型结构

```
MiniGPT
├── Token Embedding (词表 → 128维)
├── Positional Embedding (位置 → 128维)
├── Transformer Block × 6
│   ├── LayerNorm
│   ├── Causal Self-Attention (4头，带因果掩码)
│   ├── LayerNorm
│   └── FeedForward (128 → 512 → 128，GELU激活)
├── Final LayerNorm
└── Linear Head (128 → 词表大小)
```

### 关键技术细节

| 组件 | 说明 |
|------|------|
| **因果注意力** | 使用下三角掩码确保模型只能看到当前位置及之前的字符 |
| **温度采样** | `temperature=0.1` 降低随机性，使生成更确定性、更符合训练风格 |
| **Top-K 采样** | 限制候选字符为概率最高的 5 个，避免生成罕见字符 |
| **梯度裁剪** | 最大范数 1.0，稳定训练过程 |

## 快速开始

### 1. 训练模型

首次运行时会自动检测是否存在已保存的模型文件（`minigpt.pt`），若不存在则自动训练：

```bash
python LLM2.py
```

训练参数：
- 训练步数：6,000 steps
- 批次大小：16
- 序列长度：16
- 学习率：1e-3
- 优化器：AdamW

训练过程中每 500 步打印一次损失值，训练完成后模型自动保存。

### 2. 加载已有模型

若 `minigpt.pt` 已存在，程序会自动加载并跳过训练阶段。

### 3. 交互生成

模型就绪后，输入任意诗歌中的汉字作为开头，模型将续写后续内容：

```
模型已就绪，输入文字与它对话（输入q退出）

你：春眠
LLM：春眠不觉晓，处处闻啼鸟。

你：床前
LLM：床前明月光，疑是地上霜。

你：白日
LLM：白日依山尽，黄河入海流。
```

输入 `q`、`quit` 或 `exit` 退出程序。

## 使用示例

### 命令行交互

```bash
$ python LLM2.py
模型已就绪，输入文字与它对话（输入q退出）

你：千山
LLM：千山鸟飞绝，万径人踪灭。

你：葡萄
LLM：葡萄美酒夜光杯，欲饮琵琶马上催。

你：向晚
LLM：向晚意不适，驱车登古原。
```

### 注意事项

- 输入的字符**必须**在训练语料中出现过，否则会提示字符不在词表中
- 输入长度超过 16 个字符时，模型只取最后 16 个作为上下文
- 生成在逗号或句号处自动停止，保证诗句结构完整

## 模型局限性

1. **语料规模有限**：仅 24 首古诗，模型无法生成训练集以外的诗句
2. **记忆倾向强**：低温度采样使模型倾向于复现训练数据中的原句
3. **无现代汉语能力**：词表仅包含古诗中的汉字，无法处理现代文本
4. **上下文窗口短**：16 个字符的上下文限制了对长诗的理解

## 扩展建议

- 扩充训练语料至数百/数千首古诗，提升泛化能力
- 引入分词或基于拼音的建模，增强韵律控制
- 添加条件生成（如指定诗体、主题、韵脚）
- 增大模型规模（层数、维度、注意力头数）
- 实现 Beam Search 或 Nucleus Sampling 提升生成多样性
